Link Library 6.0 Public Beta

May 19th 2018 Update

When is the official version of Link Library 6.0 going to be released?

My current plan is to release Link Library 6.0 on June 17th 2018.

Should I update right away when it becomes available?

The beta version has now been available for 5 months. I have corrected a number of issues with the plugin over this period, leading to the current Beta 12 version. I believe that the plugin will be stable for most users. However, it is possible that some combination of options has not been exercised during the beta and that there will be some issues when all users start getting the update. If you want to be sure that the new version is stable, perhaps wait a bit. I will update the version number to 6.1 once I feel that I have stopped receiving a lot of support questions related to the updae.

Why has this beta cycle taken so long?

I have said it before and I will say it again. I am a single person developing this plugin, and I travel a lot for work. Almost every other week for the past 4 months. I did not want to release version 6.0 to the general public and leave users hanging if they encountered major issues. After June 17th, I should be home for 3-4 weeks, which will give me the time to answer questions that come up around the new version.

As you may have seen at the top of your web site’s admin pages, Link Library is about to take a big leap forward by converting to using Custom Post Types instead of relying on the legacy WordPress Link Manager. It took me 3 years to get to this point, between a busy work with lots of travel, various custom WordPress plugin development projects and life.

The new version brings a number of improvements:

  • New link editor: Instead of working in the Links section of the admin, users work in the Link Library section of the admin, where they are presented with a new editor only containing relevant fields. I will have much more control over the code base of this editor moving forward.
  • Display pages for links: Instead going directly outside your site, links can be displayed as a page on your site. A templating system allows you to display the link contents, as well as insert links address along with other link information in the page
  • Paged link list: One of the biggest user complaints is that all links were displayed at once by the old Link Manager. Link Library now offers the ability to see links using a paged interface, just like posts or pages, which allows users to navigate their links even if they have hundreds or thousands.
  • Hierarchical categories: It is now possible to create hierarchies of link categories.

Now, all of this being said, the move to Custom Post Types also means that other plugins that used to interact with your links will no longer work. According to the user survey I ran a while back, only a very small fraction of users use Link Library with other plugins. If this is the case, feel free to contact me via the support forum on wordpress.org to report a issue on the functionality that was provided by other plugins that you no longer have access to and I will look into bringing it back.

Beta testers needed!!

With a plugin as large as Link Library, testing can be quite daunting. This is why I am looking for people to test the new release of Link Library and provide feedback on any issue they encounter. You should normally be able to drop Link Library in place and get your site up and running as if nothing had changed.

Before installing Link Library 6.0

  • Do a backup of your site database
  • Backup the link-library folder via FTP from your site’s wp-content/plugins directory
  • If you have a staging version of your site where you can make changes without risking taking down your production site, it might be best to use this version

Download and install Link Library

  • Download the beta 12 version of Link Library 6.0 here.
  • Delete all files from your site wp-content/plugins/link-library directory
  • Extract beta plugin files to the directory
  • Back in your WordPress site admin, click on Dashboard to refresh the Interface. You should see a new item called Link Library.

 

 

 

 

 

 

 

 

  • The old Links menu still appears in the interface for the moment. I will hide it once Link Library 6.0 becomes the official version. I left it for now to allow you to easily check on values from old links.

Check your site

  • Visit pages that display links using Link Library and see if they are displaying the right content and if the links are displayed as expected.

Report issues (and lack thereof)

  • If you run into issues, please report issues using my the comment section below, in addition to my contact form
  • Please be as detailed as possible in detailing the issue
  • If possible, please attach images showing the problem(s)
  • If possible, leave the new plugin in place and activate Debug Mode under the Link Library Global Options section so I can troubleshoot issues live
  • If you need to get back to a working condition, you can just delete the new version and restore the old one. All configuration options are compatible between the two versions.
  • If there are problems with the rendering of the links page, please send me the HTML code of the page. It would also be useful if you can then revert to the previous version of Link Library and save the old HTML code so that I can compare and understand the issue
  • Finally, it would be useful if you can export your library configuration so I can load it up on my side and reproduce your issues
  • If you have a smooth migration and no problems, please let me know as well! It would help to know how close I am to release quality.

Next steps

Based on the feedback I get, I will either release the plugin, or fix issues and release further beta versions on this page. Since I travel a lot for work, this process might take a few weeks to a few months, but I really want to get this new version out the door!

If you want to be notified on new beta versions, sign up for the mailing list.

      For paged view, you need to go under Library Configurations, under the Advanced tab, then find Link Name in the table. In that row, change the Link Source from ‘Primary’ or ‘Secondary’ to ‘Dedicated Page’. Your links will now redirect to individual local pages on your site.       If you want to create a custom template for this single item page, you can do so under the Single Item Layout tab in Global Options. The default value for this field is just to show [the_content]. You can use a number of different tags in this template to create a more custom layout. For example, create a page header with the link name, image and description, followed by the full-page content of the editor. Or vice-versa. Current tags that are available are:

      – [link_title]
      – [link_content]
      – [link_description]
      – [link_large_description]
      – [link_image]
      – [link_price]
      – [link_price_or_free]
      – [link_address]
      – [link]
